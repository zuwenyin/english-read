# 阶段六：首页 + 个人中心（前后端打通）

> **思路**：将原来的 Phase 9（首页 + 个人中心）独立为一个阶段。
> 依赖 stats/recent API（已在 Phase 5 实现），此阶段专注前端页面。
> 完成后即可看到完整的面貌——首页选择年级/查看学习概览，个人中心查看统计和管理账号。

---

## Task 6.1：实现首页（/）

### 首页两种状态

**状态一：未选择年级（首次访问 / 未登录）**

- Hero 区块：欢迎语 + 平台简介
- 4 张年级选择卡片（小学📘 / 初中📗 / 高中📕 / 大学📙）
- 卡片 hover 微缩放动画
- 选择后存储到 `localStorage.selectedLevel`

**状态二：已选择年级**

- 学习概览区：调用 `GET /api/stats/overview`
  - 已学单词数 / 已读文章数 / 本周学习时长
- 推荐词书：调用 `GET /api/word-books?level=xxx`
- 推荐文章：调用 `GET /api/articles?level=xxx&pageSize=6`

### 执行步骤

1. 实现 `src/pages/Home.tsx`：
   - 判断 `localStorage.selectedLevel` 是否存在
   - 分别渲染两种状态
   - react-query 缓存统计数据（`staleTime: 30000`）

### 验收标准

- [ ] 未选择年级时显示 4 张卡片
- [ ] 选择年级后显示学习概览
- [ ] 刷新后年级选择不丢失
- [ ] 统计数据正确
- [ ] 推荐词书和文章正确显示

---

## Task 6.2：实现个人中心页（/profile）

### 执行步骤

1. 实现 `src/pages/Profile.tsx`：
   - **学习统计区**：调用 `GET /api/stats/overview`
     - 已学单词数、已读文章数、平均成绩
   - **最近学习记录**：调用 `GET /api/progress/recent`
     - 最近词书（3 条）+ 最近文章（3 条）
   - **账号信息区**：
     - 显示用户名（不可修改）、邮箱（可修改）
     - 修改邮箱表单（antd Form，提交 `PUT /api/auth/profile`）
     - 修改密码表单（antd Form，需输入新密码）
   - **退出登录按钮**

### 验收标准

- [ ] 学习统计正确显示
- [ ] 最近学习记录正确
- [ ] 修改邮箱成功
- [ ] 修改密码成功
- [ ] 退出登录清除状态并跳转
- [ ] 表单校验正确

---

## 阶段总结

**此阶段完成后，整个应用的面貌就完整了：**

```
首页（/）
  ├── 未登录 → 4 张年级选择卡
  └── 已登录+已选年级 → 学习概览 + 推荐词书 + 推荐文章

个人中心（/profile）
  ├── 学习统计（单词数 / 文章数 / 平均成绩）
  ├── 最近学习记录
  └── 账号信息管理（邮箱 / 密码 / 退出）
```

**此时系统已有完整的 9 个页面，可以走通完整的端到端流程：**

```
注册 → 登录 → 选择年级 → 首页概览
                       → 词书市场 → 单词学习 → 单词测试
                       → 阅读列表 → 文章阅读 → 答题
                       → 个人中心 → 查看统计 → 管理账号
```
